# Agent Handbook — SIMP

This directory contains ready-to-customize playbooks for AI agents collaborating on the SIMP repository.

## Available Agents (SIMP)

- [Code Reviewer](./code-reviewer.md) — Review code changes for quality, style, and best practices
- [Bug Fixer](./bug-fixer.md) — Analyze bug reports and error messages
- [Refactoring Specialist](./refactoring-specialist.md) — Identify code smells and improvement opportunities
- [Test Writer](./test-writer.md) — Write comprehensive unit and integration tests (Vitest)
- [Documentation Writer](./documentation-writer.md) — Create clear, comprehensive documentation
- [Performance Optimizer](./performance-optimizer.md) — Identify performance bottlenecks
- [Security Auditor](./security-auditor.md) — Identify security vulnerabilities
- [Frontend Specialist](./frontend-specialist.md) — Design and implement user interfaces (React, shadcn)
- [Architect Specialist](./architect-specialist.md) — Design overall system architecture and patterns
- [Devops Specialist](./devops-specialist.md) — Design and maintain CI/CD pipelines
- [AI Solutions Architect](./ai-solutions-architect.md) — Architect logic and rules for AI-assisted workflows (optional reference)

## Reference only (not specific to SIMP stack)

- [Workflow Developer](./workflow-developer.md) — n8n workflows (not used in SIMP; reference only)
- [AI QA Engineer](./ai-qa-engineer.md) — Behavioral QA for conversational agents (optional reference)
- [AI Debug Specialist](./ai-debug-specialist.md) — Conversation log analysis (optional reference)

## How To Use These Playbooks

1. Pick the agent that matches your task.
2. Enrich the template with project-specific context or links (see Key Files / Repository Starting Points in each playbook).
3. Share the final prompt with your AI assistant.
4. Capture learnings in the relevant documentation file so future runs improve.
